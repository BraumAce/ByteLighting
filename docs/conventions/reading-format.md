# 阅读记录格式规范

`src/program/reading/{year}/{year}.{month}.md` 下的月度阅读记录遵循统一格式，便于读者快速扫读、判断是否点开原文。

## Frontmatter

```yaml
---
title: "{year}.{month} 期"
order: -{month}             # 负序，使新月份排在前面
category:
  - 文章
---
```

正文起始：

```markdown
# {year}.{month} 期

<!-- more -->
```

## 单条记录格式

每条阅读记录采用「标题 + 简短描述」的内联格式：

```markdown
- [文章标题](文章链接)：{一段简短的描述，让人一下就能知道这篇文章在讲什么}
```

### 描述写作要点

- **一句话原则**：用一句完整的话点明文章核心，不分段、不分点
- **信息密度高**：开门见山给出文章的方法论 / 核心观点 / 关键路径
- **辅助判断**：让读者扫一眼就能决定是否点开原文细读
- **风格一致**：技术术语保留英文（如 AGENTS.md、Harness、SDD），中文表述自然
- **避免空话**：不写「这是一篇好文章」「值得一读」等无信息量的修饰语

### 示例

✅ 推荐：

```markdown
- [一个文件让 AI Coding 效率翻倍：AGENTS.md 实践指南](https://...)：怎么写好一份「给 AI 看的 README」——遵循「地图，而非手册」原则，配合仓库聚合、验证闭环、lint 自动化、参考项目源码引入，让 AI「打开即理解、改完即验证」。
```

❌ 不推荐（过于冗长、分段）：

```markdown
- [一个文件让 AI Coding 效率翻倍：AGENTS.md 实践指南](https://...)

  阿里巴巴工程师基于 Spring Boot + React 管控系统的实战经验……（多段长文）
```

❌ 不推荐（无信息量）：

```markdown
- [一个文件让 AI Coding 效率翻倍：AGENTS.md 实践指南](https://...)：介绍了 AGENTS.md 的实践经验，值得一读。
```

## 日期分组

同月内按日期降序分组（最新日期在最前），使用 `## {month}.{day}` 作为小标题：

```markdown
## 5.6

- [文章 A](url)：描述...
- [文章 B](url)：描述...

## 5.4

- [文章 C](url)：描述...
```

未明确日期或处于月初汇总的链接，可不带 `##` 标题，直接放在正文起始。

## 协作操作

当作者贴入若干文章链接，让 AI 协助补充描述时：

1. 抓取每篇文章的正文内容（微信公众号链接走 `mcp__plugin_everything-claude-code_exa__web_fetch_exa` 等可绕过验证的工具，WebFetch 通常会被反爬挡住）
2. 提炼每篇文章的核心方法论 / 关键路径 / 实测结论
3. 按上述「单条记录格式」补全描述，保持风格一致
4. 不擅自调整原有的链接顺序与日期分组
