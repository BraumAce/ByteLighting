# Git 提交规范

双轨制：写作类用中文冒号，工程类走 Conventional Commits。

## 内容更新轨（中文冒号 `：`）

阅读记录、文章、读书笔记等写作类改动：

- `add：{主题}` — 新增
- `update：{主题}` — 更新

主题为阅读期号（`{year}.{month}`）或文章 / 主题名。

```
add：2026.5
update：Agent Skill 设计哲学
```

## 工程类轨（英文冒号 `:`）

目录结构、配置、依赖、构建、规范文档等工程类改动：

| 类型 | 用途 |
| --- | --- |
| `feat:` | 新功能 / 大型内容里程碑 |
| `fix:` | 修复 |
| `refactor:` | 重构 |
| `chore:` | 杂项整理 |
| `docs:` | 工程文档（区别于博客文章） |

```
refactor: 重塑博客目录结构，回归扁平博客形态
chore: 清理 frontmatter 与文档，回归博客 schema
```

## 描述写法

- 动宾结构，中文为主，技术术语保留英文
- 并列改动用「，」分隔，补充说明用「—」或「+」
- 不附 `Co-Authored-By`、`Generated with` 等尾部信息
