# 软件系统环境与工具

## 1. 软件开发环境

1. 软件开发环境（Software Development Environment，SDE），应支持多种集成机制
2. 定义：是支持软件工程化开发和维护的一组软件工具和 **环境集成机制**
3. 要求：需要支持 <u>平台</u>、<u>数据</u>、<u>界面</u>、<u>控制</u> 和 <u>过程集成</u>，且支持小组工作方式，提供配置管理
4. 主要功能：
    1. 维护软件开发的一致性与完整性
    2. 实现配置管理及版本控制
    3. 支持数据的多种表示形式及自动转换
    4. 实现信息的自动检索与更新
    5. 提供项目控制和管理功能
    6. 支持开发方法学
5. 集成机制组成：
    1. **<u>环境信息库</u>**：存储 <u>与系统开发相关</u> 的信息，是 SDE 的核心部分，支持多工具间的 <u>信息交流与共享</u>
    2. **<u>过程控制与消息服务器</u>**：实现 <u>过程集成和控制集成</u> 的基础，通过控制流程、调度工具及传递消息来协调各个开发活动
    3. **<u>环境用户界面</u>**：包括环境总界面以及各部件各工具的界面。具有一致的用户界面，可以发挥环境的优越性，提高工具的使用效率，减轻用户的学习负担

## 2. 软件系统工具

软件系统工具通常可以按 <u>软件过程活动</u> 将软件工具分为软件开发工具、软件维护工具、软件管理和软件支持工具。

### 2.1 软件开发工具

1. **需求分析工具**：分为 <u>基于自然语言或图形描述的工具</u>、<u>基于形式化需求定义语言</u> 的工具
2. **设计工具**：用于软件设计阶段，辅助建立系统结构、模块分解、接口设计等（架构，详细）
3. **建模工具**：用于 <u>辅助</u> 建立软件系统的抽象模型
4. **编码与排错工具**
5. **测试工具**

### 2.2 软件维护工具

1. **<u>版本控制工具</u>**
2. 文档分析工具
3. 开发信息库工具
4. 逆向工程工具
5. 再工程工具

### 2.3 软件管理工具

1. 项目管理工具
2. 配置管理工具（也可能包含 **<u>版本控制工具</u>**）

### 2.4 软件支持工具

1. **软件评价工具**
2. 软件开发工具的评价和选择

## 2.5 软件配置管理（SCM）工具

1. 概念：是一种在软件生命周期中对软件配置项进行标识、组织、控制和跟踪的技术与方法，确保软件产品的一致性与可追溯性。
2. 功能：<u>版本控制</u>、<u>变更管理</u>、<u>配置状态管理</u>、<u>访问控制</u> 和 <u>安全控制</u>。
